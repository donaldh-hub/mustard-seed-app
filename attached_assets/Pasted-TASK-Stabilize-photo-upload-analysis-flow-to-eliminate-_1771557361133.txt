CONTEXT / SYMPTOMS (confirmed):
- User selects an image, sees local preview, then upload fails with “Failed to fetch”.
- Retrying multiple times still fails.
- Some smaller images/screenshots may work; camera photos or larger images often fail.
- We need reliable uploads across mobile + desktop.

GOALS:
1) Upload must NOT hang or crash on large images; it must return a clear JSON error if limits are exceeded.
2) Upload must succeed consistently by:
   - enforcing server-side multipart limits
   - compressing/resizing client-side before upload
   - adding a server-side timeout and abort logic around AI/image analysis
   - ensuring CORS/headers are correct
   - ensuring fetch always receives JSON (success or failure)
3) Do not award water if analysis fails or upload fails.
4) Save the photo memory ONLY after upload succeeds and analysis returns valid result.
5) Provide visible user feedback: uploading → analyzing → success OR a clear actionable error.

- Add per-step timeouts: upload persistence, AI analysis, DB writes.
- Wrap the entire route in try/catch and ALWAYS respond with JSON:
  { ok: true, ... } or { ok: false, errorCode, message }
- Add an error middleware to normalize multer + payload errors:
  - LIMIT_FILE_SIZE
