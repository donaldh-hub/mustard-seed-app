  PAYMENT_FAILED,
  CANCELED_PENDING_EXPIRATION

STATE MACHINE RULES:

1) On App Launch:
   - Verify receipt with store
   - Compare store expiration date with Firebase
   - If mismatch → update Firebase to match store
   - Store is source of truth

2) Trial Logic:
   - Trial begins only after store confirms trial subscription purchase
   - trialEndsAt = purchaseDate + 14 days
   - After trialEndsAt:
       if subscription renewed → PREMIUM_ACTIVE
       else → LITE

3) Cancellation:
   - If user cancels:
       state = CANCELED_PENDING_EXPIRATION
       retain Premium until expiration date
   - After expiration → LITE

4) Payment Failure:
   - If store signals billing issue:
       state = PAYMENT_FAILED
       enter grace period if supported
   - If resolved → PREMIUM_ACTIVE
   - If not resolved after expiration → LITE

---

PROFILE BADGE LOGIC:

Display based strictly on subscriptionState:

LITE → "Lite"
PREMIUM_TRIAL_ACTIVE → "Premium (Trial)"
PREMIUM_ACTIVE → "Premium"
PREMIUM_GRACE_PERIOD → "Premium (Grace)"
PAYMENT_FAILED → "Payment Issue"
CANCELED_PENDING_EXPIRATION → "Premium (Ends Soon)"

Badge must:
- Auto-update on state change
- Never rely on cached local state
- Pull from Firebase listener

FEATURE GATING RULE:

Premium Features unlocked only if:

subscriptionState IN:
  - PREMIUM_TRIAL_ACTIVE
  - PREMIUM_ACTIVE
  - PREMIUM_GRACE_PERIOD
  - CANCELED_PENDING_EXPIRATION (before expiration)

Else → locked
